   DOM

   DOM is a programming interface for web documents. It represents the structure of a document as a tree of objects, where each object corresponds to a part of the document, such as elements, attributes, and text. Key concepts of the DOM:

  • Document: The top-level object that represents the entire HTML or XML document.
  • Element:Represents an HTML or XML element in the document. Elements may contain attributes, text, and other elements.
  • Nodes:The base type for all objects in the DOM. Nodes can be elements, text, comments, etc.
  • Attribute: Represents an attribute of an element.
  • Text: Represents the textual content within an element.

Doctype in HTML

1. Definition :

  • The HTML document type declaration, commonly known as DOCTYPE, is an essential instruction placed at the beginning of every HTML or XHTML document.
  • It serves as an informational tag for web browsers, informing them about the version of HTML or XHTML used in the document.
  • The DOCTYPE declaration is not an HTML element or tag; rather, it guides browsers on how to interpret the rest of the document.

2. Purpose and Importance:

  • Version Specification:The DOCTYPE helps browsers understand which version of HTML (or XHTML) the website adheres to. This knowledge ensures consistent rendering across different browsers.
  • Rendering Process: By providing the correct DOCTYPE, you ease the rendering process for browsers. They can apply the appropriate rules and standards to display the content accurately.

3. HTML5 DOCTYPE

  • In modern web development, the DOCTYPE declaration for HTML5 is straightforward
  • This simple declaration indicates that the document follows the HTML5 standard.

4. Older HTML Documents:

  • In older HTML documents (such as HTML 4 or XHTML), the DOCTYPE declaration was more complex. It had to refer to a Document Type Definition (DTD) specific to that version.

    Depreated Elements

    Deprecated elements in web development refer to HTML elements that have been officially discouraged or phased out by the World Wide Web Consortium (W3C), the organization responsible for developing and maintaining web standards. Developers are advised to avoid using deprecated elements and migrate to newer, recommended alternatives to ensure better compatibility, security, and adherence to current best practices. Here are a few examples of deprecated HTML elements as of my knowledge cutoff in January 2022:

    1. font :
      • Deprecated in HTML4.01: The font element was used to define the font, color, and size of text. Instead, developers are encouraged to use CSS for styling.
    2. center :
      • Deprecated in HTML4.01: The center element was used to center-align content on a webpage. CSS should be used for layout and positioning instead.
    3. strike and s :
      • Deprecated in HTML5:These elements were used for strikethrough text. The recommended way is to use the del element with CSS for styling.
    4. acronym :
      • Deprecated in HTML5:The acronym element was used to define an acronym. The recommended replacement is the element.
    6. frame , frameset, noframes :
      • Deprecated in HTML5:Frames and framesets were used for creating a layout with multiple independent sections. Modern websites use more flexible and responsive layout techniques, such as CSS and HTML5 elements like div.
    7. applet :
      • Deprecated in HTML5: The applet element was used to embed Java applets. With the decline of Java applets and the rise of other technologies, this element is no longer supported in modern browsers

    Section Tag :

    The section tag is used to define sections in an HTML document. It is a semantic HTML5 element that helps to structure and group related content.

    Article Tag :

    The article tag in HTML5 is used to define an independent, self-contained piece of content. It is often used for articles, blog posts, forum posts, news items, and other similar content that can be distributed and reused independently.

    Aside Tag :

    The aside tag in HTML is used to define content that is tangentially related to the content around it. It is often used for sidebars or content that is considered secondary or additional to the main content of the page.

    Header Tag :

    The header tag in HTML5 is used to define the header section of a webpage. It typically includes elements such as the website's title, logo, navigation menu, and other introductory content.

    Footer Tag :

    The footer tag in HTML5 is used to define the footer section of a webpage. It typically includes information such as copyright notices, links to terms of service or privacy policies, and other footer-related content.

    Nav :

    The nav tag in HTML5 is used to define a navigation menu on a webpage. It is typically used to wrap around a list of links or navigation-related content.

    Dialog Tag :

    The dialog tag in HTML5 is used to create a dialog box or modal within a web page. It allows developers to create interactive dialogs or pop-up windows that can display information, receive user input, or perform specific actions.

    Figure Tag :

    The figure tag in HTML is used to encapsulate any content that is referenced from the main content, typically images, diagrams, illustrations, videos, code snippets, or similar content. It is often used in conjunction with the figcaption tag to provide a caption for the content within the figure.

It works in the same way that we use a paper form to gather or submit information. This lesson guides you on creating web forms and their various useful components, which allow you to collect and manage data easily and efficiently.
    A Web Forms is a part of an HTML document containing HTML form elements such as input, select, checkboxes, radio, buttons, etc. Users typically complete a form by entering texts, selecting options, and modifying form elements before submitting the form for processing.

    • Contact form

    How to create forms in HTML?
    The form tag in HTML is used to define the form in an HTML document. This tag is a container for all other input elements used within this form tag to obtain user input.
    Attributes of the HTML form Element
    The form element has two important attributes: the action and the method attributes. Here, the form action specifies the location where the form data will be submitted, and the form method specifies the HTTP method that is usually GET / POST. The various HTTP methods are:
    • GET
    • POST
    • PUT
    • PATCH
    • DELETE
    What are HTML form elements?
    Different input types, such as text fields, checkboxes, radio buttons, submit buttons, etc., are used inside the form tag to obtain user inputs such as usernames, emails, passwords, etc. According to the input field type, these inputs are manually typed or selected by the user.
