CSS

CSS (Cascading Style Sheets) is a powerful language that plays a crucial role in web development. Let’s delve into the details:

  1. What is css ?

    CSS (Cascading Style Sheets) is a powerful language that plays a crucial role in web development. Let’s delve into the details:
    • CSS stands for Cascading Style Sheets.
    • It is a style sheet language used to describe the look and formatting of a document written in a markup language (usually HTML).
    • CSS provides an additional layer of control over how web pages and user interfaces appear.
    • By using CSS, you can:
      • Control the color of text.
      • Define font styles.
      • Specify spacing between paragraphs.
      • Set background images or colors.
      • Create responsive layouts for different devices and screen sizes.
      • Apply a variety of other visual effects.

    • how does CSS work ?

      • CSS defines style rules that are interpreted by the browser and then applied to corresponding elements in your document.
      • A style rule set consists of two main parts:
        • Selector : Targets specific HTML elements to apply styles.
        • Declaration Block :Contains one or more declarations, each consisting of a property and its corresponding value.

    • Why use css ?

      • Time- Saving :Write CSS once and reuse it across multiple HTML pages
      • Easy Maintance :Make global changes by updating the style, and all elements in all webpages will automatically reflect the change.
      • Search engine Optimization (SEO) Clean CSS code helps search engines understand content better.
      • Superior Styling: CSS offers a wider array of attributes than HTML, allowing you to create visually appealing web pages.
      • Offline Browser:CSS can store web applications locally, enabling offline viewing.

History of CSS

The history of CSS (Cascading Style Sheets) is intertwined with the development of the World Wide Web and the need for improved presentation and styling of web documents. Here is a brief overview of the key milestones in the history of CSS:
  1. Early Web Design (1990s):
    • In the early days of the web, HTML (Hypertext Markup Language) was primarily used for structuring documents, and the styling was embedded directly within the HTML code.
    • This inline styling made it challenging to maintain consistency across multiple pages and devices.
    2. CSS Emerges (1994-1996):
    • The need for a separate styling language led to the development of CSS.
    • Håkon Wium Lie and Bert Bos proposed the idea of Cascading Style Sheets in October 1994. Lie worked at CERN (European Organization for Nuclear Research), where Tim Berners-Lee, the inventor of the World Wide Web, also worked.
    • In 1996, the World Wide Web Consortium (W3C) released the first official CSS specification, CSS1. This marked the beginning of a standardized approach to styling web documents.
  3. CSS Level 2 (1998):
    • CSS Level 2 (CSS2) was introduced to address limitations in the initial CSS specification.
    • CSS2 brought new features and improvements, including positioning, z-index, and support for media types.
  4. Browser Wars and Inconsistent Implementations (Late 1990s - Early 2000s):
    • During the "Browser Wars" between Internet Explorer and Netscape Navigator, there were inconsistencies in the implementation of CSS. Web developers faced challenges in creating cross-browser-compatible stylesheets.
  5. CSS Zen Garden (2003):
    • The CSS Zen Garden project, launched by Dave Shea in 2003, demonstrated the power of CSS for design by showcasing different styles applied to the same HTML markup. This project highlighted the separation of content and presentation.
  7. CSS3 and Modules (2005 Onwards):
    • The development of CSS3 began, introducing modularization to allow for the release of individual modules independently.
    • Various modules, such as Selectors, Colors, and Backgrounds, provided new features and capabilities for web designers.
  9. Responsive Web Design (2010s):
    • The growing use of smartphones and tablets led to the need for responsive web design. Media queries in CSS3 became crucial for adapting layouts to different screen sizes and devices.
  10. CSS Grid and Flexbox (2010s):
    • CSS Grid Layout and Flexbox, two layout models, were introduced to simplify the creation of complex and responsive page layouts. They provided more control over the positioning and alignment of elements.
  11. Current State (2020s):
    • CSS continues to evolve with ongoing updates and refinements.
    • Features like CSS Variables, Custom Properties, and new layout techniques contribute to more efficient and maintainable stylesheets.

Versions of CSS

As of my last knowledge update in January 2022, there were several versions of Cascading Style Sheets (CSS) that had been released. Keep in mind that newer versions may have been released since then. As of 2022, some of the major CSS versions include:

  1. CSS!(1996): The initial version of CSS designed to style and format documents on the web.
  2. CSS2 (1998): Introduced new features like absolute, relative, and fixed positioning of elements, z-index, and media types.
  3. CSS2.1 (2011): An updated version of CSS2 with some fixes and improvements. CSS2.1 aimed to bring the specification in line with implementations.
  4. css3: Unlike previous versions, CSS3 is not a monolithic specification but rather a collection of smaller modules, each introducing new features. Some of the notable CSS3 modules include:
    • Selectors Level 3 (Selectors): Enhanced selectors for more specific targeting of HTML elements.
    • Box Model: Additional properties and features for the layout and sizing of boxes.
    • Media Queries:Allows styles to be applied based on characteristics of the device, such as screen size and resolution.
    • Transitions and Animations: Introduces properties for creating smooth transitions and animations.
    • Flexbox and Grid: Layout modules for more sophisticated and responsive page layouts.
  5. css4: While there isn't a formal specification known as CSS4, the term has been informally used to refer to ongoing and proposed additions to the CSS language beyond CSS3.
  6. CSS Grid Layout Level 2 (2021): A specification that builds upon the original CSS Grid Layout, providing additional features and improvements.

What is new in css3?

As of my last knowledge update in January 2022, CSS3 is a collection of modular specifications that introduce a wide range of new features and enhancements to the Cascading Style Sheets language. Here are some key features introduced in CSS3:

  1. Selectors Level 3:Additional and more powerful selectors, allowing for more precise targeting of HTML elements.
  2. Box Model:New properties and features related to the box model, including the box-sizing property, which allows you to control how the width and height of elements are calculated.
  3. color:More advanced color options, such as RGBA (Red, Green, Blue, Alpha) and HSLA (Hue, Saturation, Lightness, Alpha), providing transparency control.
  4. Text Effects:New properties for controlling text shadow and text overflow, as well as the introduction of the text-overflow property.
  5. Web Fonts:The @font-face rule allows web designers to use custom fonts on their websites.
  6. 2D Transforms:Transformations for elements, including translation, rotation, scaling, and skewing.
  7. transitions:The transition property enables the smooth transition of property values over a specified duration.
  8. animations:The @keyframes rule and the animation property allow for the creation of complex animations directly in CSS.
  9. Multiple Column Layout:The column-count and column-gap properties enable the creation of multi-column layouts for text.
  10. flexbox:A layout model designed for the efficient distribution of space among items in a container even when their size is unknown or dynamic.
  11. Grid Layout:A powerful two-dimensional layout system for handling both columns and rows, allowing for complex and responsive page layouts.
  12. Media Queries:Allows styles to be applied based on the characteristics of the device, such as screen size, resolution, or device orientation.
  13. Responsive Design:CSS3 features, combined with media queries, transitions, and flexible grid systems, have contributed to the widespread adoption of responsive web design.

CSS Selectors and it's type

A CSS Selector is the most important part of css they are used to select element on a html page. The selected element thus can be style according to the need and demand of the user. CSS Selector are of various name :

  • Element Selector
  • ID Selector
  • Class Selector
  • Universal Selector
  • Group Selector
  • Tag Selector
  • Attribute Selector
  • Sub Selector
  1. Element Selector

    The element selector select HTML element based on the element name

    Syntax:

    Element_name{
    //CSS properties
    }
  2. ID Selector

    The id selector uses the id attribute of an HTML element to select an specific id .Write a hash(#) charecter followed by the id of the element

    Syntax :

    #id{
    // CSS property
    }
  3. Class Selector

    The class selector select html elements with a specific class attribute. You declare a CSS class by using a dot(.) followed by the class name

    Syntax :

    .class{
    //CSS property
    }
  4. Universal Selector

    The universal selector (*) in css is used to select all the element in a html document. It also includes other element which are inside under another element

    Synatx :

    *{
    //CSS property
    }
  5. Group Selector

    The selector is used to style all comma-seperated elements with the same style

    Syntax :

    #div,.para,h1{
    //CSS property
    }
